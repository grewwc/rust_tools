# debugger.skill 优化说明

## 优化背景

在之前的代码修改过程中，经常会在源代码目录创建 `.backup`、`.bak` 等临时备份文件，且任务完成后未及时清理，导致：
- 源码目录混乱
- Git diff 包含不必要的文件
- 可能误提交备份文件到版本控制

## 优化内容

### 版本升级
- **版本号**: 2.0 → 2.1
- **新增章节**: guidelines（指导原则）

### 新增指导原则

#### 1. 核心原则 (Core Principles)
- **最小改动**: 优先使用 `apply_patch` 进行精确修复，避免整文件重写
- **精准定位**: 先读取相关代码段，确认问题根因后再修改
- **验证驱动**: 每次修复后必须编译/测试验证

#### 2. 文件修改规范 (File Modification)
- 优先使用 `apply_patch` 进行行级精确修改
- 避免 `write_file` 整文件覆盖，除非重构需要
- **禁止创建 .backup、.bak、.tmp 等临时备份文件**
- 如必须备份，使用 `/tmp/` 目录且注明用途
- 修改完成后立即清理所有临时文件

#### 3. 临时文件管理 (Cleanup Policy)
- **不在源代码目录创建任何备份文件**
- 临时文件必须使用 `/tmp/` 或系统临时目录
- 工具调用失败时也要清理已创建的临时文件
- 完成任务前检查并清理所有临时文件
- 使用命令：`ls *.backup *.bak 2>/dev/null` 检查遗留文件

#### 4. 错误处理流程 (Error Handling)
- 读取完整错误信息，定位错误行号和类型
- 读取错误相关代码上下文（前后 10-20 行）
- 分析错误根因，提出最小修复方案
- 应用修复后立即编译验证
- 如修复失败，回滚并尝试替代方案

#### 5. 验证步骤 (Verification)
- 编译检查：cargo check / 相应语言的语法检查
- 单元测试：运行相关测试用例
- 回归测试：确保修复未引入新问题
- **清理检查：确认无临时文件遗留** ⭐新增

#### 6. 常见陷阱 (Pitfalls to Avoid)
- **避免在源码目录创建 .backup、.bak、.orig 等文件** ⭐重点
- 避免使用 sed -i 不带备份扩展名（某些系统会创建备份）
- 避免整文件重写导致不必要的 diff
- 避免多次重试同一错误而不分析根因
- 避免忽略编译器警告

#### 7. 最佳实践 (Best Practices)
- 使用 `apply_patch` 精确修改特定行
- 复杂修改分步进行，每步验证
- 保留修改前后的关键代码段用于对比
- **在提交前运行完整的清理检查** ⭐新增
- 记录修改原因和验证结果

### 新增示例

#### 好示例 (Good Examples)
1. **使用 apply_patch 精确修复借用错误**
   - 强调使用 apply_patch 而非整文件重写
   - 包含验证步骤
   - 确认无临时文件创建

2. **修复编译错误并清理**
   - 完整的修复流程

#### 坏示例 (Bad Examples)
1. **创建多个备份文件未清理** ⭐新增
   - 明确指出问题：在源码目录创建 .backup 文件
   - 列出具体不良行为
   - 提供改进方案：使用 /tmp/ 目录，完成后立即清理

2. **整文件重写导致大 diff** ⭐新增
   - 说明整文件重写的弊端
   - 提供改进方案：使用 apply_patch 精确修改目标行

## 使用建议

### 对 AI 助手的要求
1. 严格遵守 `file_modification` 规范
2. 优先使用 `apply_patch` 进行精确修改
3. 禁止在源码目录创建任何备份文件
4. 如需临时文件，必须使用 `/tmp/` 目录
5. 任务完成前必须执行清理检查

### 清理检查命令
```bash
# 检查源码目录是否有遗留的备份文件
find /Users/bytedance/rust_tools -type f \( -name "*.backup" -o -name "*.bak" -o -name "*.tmp" \) 2>/dev/null

# 或简单检查
ls *.backup *.bak 2>/dev/null
```

### 验证流程
每次代码修改后应执行：
1. ✅ 编译验证：`cargo check`
2. ✅ 功能验证：运行相关测试
3. ✅ 清理检查：`find ... -name "*.backup" ...`
4. ✅ 确认无临时文件遗留
